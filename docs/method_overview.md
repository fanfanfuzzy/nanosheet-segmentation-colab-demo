# Method Overview / 手法の概要

## Synthetic Data Generation / 合成データ生成

Synthetic nanosheet images are generated by:

1. Creating a uniform gray background (mimicking TEM background).
2. Placing random polygons (4-6 vertices) as nanosheet shapes.
3. Applying Beer-Lambert-style attenuation for realistic transparency.
4. Adding Gaussian noise to simulate detector noise.
5. Applying Gaussian blur to simulate limited resolution.

Difficulty levels control:
- **Easy**: High contrast, low noise, few nanosheets, minimal overlap.
- **Mid**: Moderate contrast and noise, some overlap.
- **Hard**: Low contrast, high noise, dense overlap.

## Zero-Shot Baseline / ゼロショットベースライン

The zero-shot baseline uses classical image processing:

1. **Adaptive thresholding** to detect dark regions.
2. **Morphological operations** (opening/closing) to reduce noise.
3. **Distance transform + thresholding** to find seed points.
4. **Connected component analysis** to separate instances.

This approach requires **no training data** but cannot adapt to
task-specific patterns.

## Trained Model (Simulated) / 学習済みモデル（シミュレーション）

In this demo, trained-model predictions are precomputed from
synthetic data only. In practice, this would be a YOLO-seg or
Mask R-CNN model trained on synthetic nanosheet images.

The key educational point: a model trained on task-specific data
can learn patterns that generic image processing cannot capture.

## Evaluation Metrics / 評価指標

| Metric | Description |
|--------|-------------|
| instance_recall | Fraction of GT instances found by predictions (IoU ≥ 0.5) |
| instance_precision | Fraction of predictions matching a GT instance |
| instance_f1 | Harmonic mean of recall and precision |
| mean_best_iou | Average of the best IoU for each GT instance |
| semantic_iou | Pixel-level foreground IoU (ignores instance boundaries) |

### Important Note on Partial GT / 部分的GTに関する注意

When ground-truth annotations are incomplete (common in real data):
- **precision** may be underestimated (correct predictions counted as false positives)
- **instance_recall** and **mean_best_iou** are more reliable primary metrics

This is why this demo uses complete synthetic GT for fair evaluation.
